TC-127 Double decimal point accepted in numeric input Fixed P2 Both ▸

Reporter: Ana P. · Filed: Jan 28, 2026 · Fixed by: frontend-team · Deployed: Mar 1, 2026

Description

Entering a number with multiple decimal points (e.g., pressing 3 . 1 . 4) produces an invalid number '3.1.4' on the display. No validation prevents additional decimal point entry.

Steps to Reproduce

1. Press 3. 2. Press decimal (.). 3. Press 1. 4. Press decimal (.) again. 5. Press 4. 6. Display shows '3.1.4'.

Expected Result

Second decimal press should be ignored.

Actual Result

Display shows '3.1.4'. Subsequent operations produce NaN.

Resolution: Added decimal existence check in inputDecimal(). Second decimal input is now ignored if currentInput already contains a period.

Deploy note: Fix verified in staging. Deployed with build 2.1.5 on Mar 1.

Linked test: SM-009 updated to include double-decimal scenario
